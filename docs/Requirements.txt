Restaurant POS System - Requirements Document

Project: Point of Sale System
Version: 1.0
Date: January 20, 2026

================================================================================

1. OVERVIEW

A desktop Point of Sale system for restaurants with two applications:
- Cashier App: Take orders and process payments
- Kitchen Display: View and update order status

================================================================================

2. FUNCTIONAL REQUIREMENTS

2.1 Order Management
- Create orders with three types: Dine-in, Parcel (takeaway), Food Delivery
- Dine-in orders require table number
- Add menu items to cart with quantity tracking
- Calculate order totals automatically
- Save orders to database

2.2 Payment Processing
- Display order summary before payment
- Accept cash payments
- Calculate change due
- Print customer receipt
- Print kitchen ticket

2.3 Kitchen Display
- Show all active orders in real-time
- Display order details: ID, type, table, items, time
- Update order status (Ordered → Processing → Done)
- Color-code orders by status
- Auto-refresh every 5-10 seconds

2.4 Menu Management
- Display active menu items with name and price
- Organize items by category
- Support active/inactive status for items

================================================================================

3. NON-FUNCTIONAL REQUIREMENTS

3.1 Performance
- Menu loads in under 1 second
- Order creation completes in under 2 seconds
- Support 100+ concurrent orders

3.2 Usability
- Modern, clean interface
- Touch-friendly buttons (100x100px minimum)
- Clear error messages
- Minimal clicks for common tasks

3.3 Reliability
- 99% uptime during business hours
- Automatic database connection retry
- No data loss on errors

3.4 Security
- Database credentials in config file (not in code)
- SQL injection prevention
- Input validation on all fields

================================================================================

4. SYSTEM REQUIREMENTS

4.1 Software
- Windows 10/11 (64-bit)
- MySQL 8.0+
- .NET 8.0 Desktop Runtime

4.2 Hardware
- Standard desktop/laptop computer
- Receipt printer (Windows compatible)
- Kitchen printer (Windows compatible)
- Network connection for database

================================================================================

5. DATABASE

5.1 Tables
menu_items: id, name, description, price, category, is_active
orders: id, order_type, table_number, status, total_amount, created_at
order_items: id, order_id, menu_item_id, quantity, price

5.2 Order Types
- DineIn (requires table_number)
- Parcel (table_number = null)
- Foodpanda (table_number = null)

5.3 Order Status
- Ordered (new order)
- Processing (kitchen working on it)
- Done (ready for customer)

================================================================================

6. USER ROLES

Cashier: Create orders, process payments
Kitchen Staff: View orders, update status
Manager: (Future) View reports, manage menu
